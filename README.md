# Checklist App

A touch-friendly step-by-step checklist app built with **Qt 6.8 / Qt Quick / C++**.
Runs natively on Raspberry Pi 4 (aarch64) and in any modern browser via WebAssembly.

## Use in the Browser (No Compilation Required)

### Your own checklists in 4 steps

1. **Fork** this repository on GitHub
2. Add your `.csv` files to the `checklists/` folder
3. Add any images referenced by your CSVs to `checklists/` too
4. `git push` — GitHub Actions generates `index.json` and deploys in ~10 seconds

Enable GitHub Pages in your fork:
> **Settings → Pages → Source: GitHub Actions**

Your fork is then live at `https://yourusername.github.io/checklist`.
**You never need to compile anything.**

## CSV Format

Five semicolon-separated columns: `text ; emoji ; image ; timer ; auto`

```
text of the step ; 🔵 ; photo.jpg ; 30s ; auto
```

| Column | Required | Description | Example |
|--------|----------|-------------|---------|
| text   | ✓ | Step text. Supports `**bold**`, `*italic*`, `\n` line breaks, emojis | `Mix well 🥣\n**30 seconds**` |
| emoji  |   | A single emoji that orbits the screen after 30 s | `🔥` |
| image  |   | Image filename (relative to `checklists/`) | `kneading.jpg` |
| timer  |   | Countdown in seconds | `30s` |
| auto   |   | Auto-advance when timer expires | `auto` |

All separators are required even if a field is empty:

```csv
Welcome!;;;;
Knead the dough;🤌;kneading.jpg;600s;
Let it rise;⏳;;3600s;auto
Bake at 200 °C;🔥;;1500s;auto
Done — enjoy!;🎉;;;
```

## Recommended Stack (Native Development)

| Layer | Version | Notes |
|-------|---------|-------|
| Qt | **6.8.3 LTS** | Full QtQuick + QtNetwork; LTS until 2026 |
| Raspberry Pi OS | **Trixie (Debian 13)** | Ships Qt 6.8.x via `apt` |
| Dev OS | **Ubuntu 24.04 LTS** | Install Qt 6.8.3 via `aqtinstall` |
| Compiler | **GCC 13** | Same ABI on both machines |

### Setup: Ubuntu 24.04 (WSL2)

```bash
sudo apt install -y cmake ninja-build gcc g++ python3-pip \
    libgl1-mesa-dev libgles2-mesa-dev libx11-dev libxkbcommon-dev

pip3 install aqtinstall
aqt install-qt linux desktop 6.8.3 gcc_64 \
    --modules qtquick qtquickcontrols2 qtimageformats \
    --outputdir ~/Qt

echo 'export CMAKE_PREFIX_PATH=~/Qt/6.8.3/gcc_64' >> ~/.bashrc && source ~/.bashrc
```

### Setup: Raspberry Pi 4 (RPi OS Trixie)

```bash
sudo apt install -y qt6-base-dev qt6-declarative-dev \
    qml6-module-qtquick qml6-module-qtquick-controls \
    qml6-module-qtquick-layouts qml6-module-qtquick-templates2 \
    cmake ninja-build gcc g++

qmake6 --version   # should print 6.8.x
```

### Build & Run

```bash
# Dev machine or Pi
cmake -B build -G Ninja -DCMAKE_BUILD_TYPE=Release
cmake --build build

# Run (X11/Wayland desktop)
./build/checklist

# Run full-screen headless (Pi without desktop)
./build/checklist -platform eglfs

# Place CSV files alongside the binary (or in the working directory)
cd /path/to/checklists && /path/to/build/checklist
```

## WebAssembly Build (Maintainers Only)

Forks never need to build WASM. If you modify C++/QML and need to update the
binary, trigger the **Build WASM** workflow manually from GitHub Actions, or push
to `main` with changes in `src/`, `qml/`, or `CMakeLists.txt`.

To build locally:

```bash
# Install Emscripten 3.1.50 (required by Qt 6.8)
git clone https://github.com/emscripten-core/emsdk.git ~/emsdk
~/emsdk/emsdk install 3.1.50 && ~/emsdk/emsdk activate 3.1.50
source ~/emsdk/emsdk_env.sh

# Install Qt WASM via aqtinstall
aqt install-qt linux desktop 6.8.3 wasm_singlethread \
    --modules qtquick qtquickcontrols2 \
    --outputdir ~/Qt

# Build
cmake -B build-wasm \
    -DCMAKE_TOOLCHAIN_FILE=~/Qt/6.8.3/wasm_singlethread/lib/cmake/Qt6/qt.toolchain.cmake \
    -DQT_HOST_PATH=~/Qt/6.8.3/gcc_64 \
    -DCMAKE_BUILD_TYPE=MinSizeRel \
    -G Ninja
cmake --build build-wasm

# Commit the artifacts
cp build-wasm/checklist.js build-wasm/checklist.wasm build-wasm/qtloader.js wasm/
```

> **Why singlethread?** GitHub Pages cannot set the `COOP`/`COEP` headers
> required by the multithreaded WASM variant. Single-thread works in all browsers
> with no server configuration.
