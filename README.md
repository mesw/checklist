# Checklist App

A touch-friendly step-by-step checklist app built with **Qt 6.8 / Qt Quick / C++**.
Runs natively on Raspberry Pi 4 and in any modern browser via WebAssembly.

## Using Your Own Checklists (No Compilation)

### 1 — Fork the repository

Click **Fork** on GitHub. You never need to compile anything.

### 2 — Add your CSV files

Drop `.csv` files (and any images they reference) into the `checklists/` folder.

### 3 — Regenerate index.json

Run the included script once after adding or removing CSVs:

```bash
./generate_index.sh
```

This scans `checklists/*.csv` and writes `checklists/index.json` automatically.
Commit the result.

### 4 — Enable GitHub Pages

In your fork: **Settings → Pages → Source → Deploy from a branch → main / (root)**

### 5 — Push and visit

```bash
git add checklists/
git commit -m "add my checklists"
git push
```

Your fork is live at `https://yourusername.github.io/checklist/wasm/checklist.html`
within a minute (GitHub Pages propagation time, no build step).

## CSV Format

Five semicolon-separated columns: `text ; emoji ; image ; timer ; auto`

| Column | Required | Description |
|--------|----------|-------------|
| text   | ✓ | Step text. Supports `**bold**`, `*italic*`, `\n` line breaks, emojis |
| emoji  |   | Single emoji that orbits the screen after 30 s |
| image  |   | Image filename (must be in `checklists/`) |
| timer  |   | Countdown e.g. `30s`, `600s` |
| auto   |   | Write `auto` to advance automatically when timer expires |

All five separators are always required even for empty fields:

```csv
Welcome! 👋\nThis is step one.;;;;
Knead the dough;🤌;kneading.jpg;600s;
Let it rise;⏳;;3600s;auto
Bake at 200 °C;🔥;;1500s;auto
Done — enjoy!;🎉;;;
```

### After editing CSVs, always run:

```bash
./generate_index.sh
git add checklists/
git commit -m "update checklists"
git push
```

## Native Development (Raspberry Pi / Linux)

### Recommended Stack

| Layer | Version | Notes |
|-------|---------|-------|
| Qt | **6.8.3 LTS** | Full QtQuick + QtNetwork |
| Raspberry Pi OS | **Trixie (Debian 13)** | Ships Qt 6.8.x via `apt` |
| Dev OS | **Ubuntu 24.04 LTS** | Install Qt 6.8.3 via `aqtinstall` |
| Compiler | **GCC 13** | Same ABI on both machines |

### Setup: Ubuntu 24.04 (WSL2)

```bash
sudo apt install -y cmake ninja-build gcc g++ python3-pip \
    libgl1-mesa-dev libgles2-mesa-dev libx11-dev libxkbcommon-dev

pip3 install aqtinstall
aqt install-qt linux desktop 6.8.3 gcc_64 \
    --modules qtquick qtquickcontrols2 qtimageformats \
    --outputdir ~/Qt

echo 'export CMAKE_PREFIX_PATH=~/Qt/6.8.3/gcc_64' >> ~/.bashrc && source ~/.bashrc
```

### Setup: Raspberry Pi 4 (RPi OS Trixie)

```bash
sudo apt install -y qt6-base-dev qt6-declarative-dev \
    qml6-module-qtquick qml6-module-qtquick-controls \
    qml6-module-qtquick-layouts qml6-module-qtquick-templates2 \
    cmake ninja-build gcc g++

qmake6 --version   # should print 6.8.x
```

### Build & Run

```bash
cmake -B build -G Ninja -DCMAKE_BUILD_TYPE=Release
cmake --build build

# Desktop / WSL with X11
./build/checklist

# Raspberry Pi headless (no desktop compositor)
./build/checklist -platform eglfs
```

Place your `.csv` files and images in the same directory you launch the binary from.

## Rebuilding the WASM Binary (Maintainers Only)

Only needed when C++ or QML source files change. Forks never need to do this.

```bash
# 1. Install Emscripten 3.1.50 (required by Qt 6.8)
git clone https://github.com/emscripten-core/emsdk.git ~/emsdk
~/emsdk/emsdk install 3.1.50 && ~/emsdk/emsdk activate 3.1.50
source ~/emsdk/emsdk_env.sh

# 2. Install Qt WASM
pip3 install aqtinstall
aqt install-qt linux desktop 6.8.3 wasm_singlethread \
    --modules qtquick qtquickcontrols2 qtimageformats \
    --outputdir ~/Qt

# 3. Build
cmake -B build-wasm \
    -DCMAKE_TOOLCHAIN_FILE=~/Qt/6.8.3/wasm_singlethread/lib/cmake/Qt6/qt.toolchain.cmake \
    -DQT_HOST_PATH=~/Qt/6.8.3/gcc_64 \
    -DCMAKE_BUILD_TYPE=MinSizeRel \
    -G Ninja
cmake --build build-wasm

# 4. Update wasm/ and commit
cp build-wasm/checklist.html \
   build-wasm/checklist.js   \
   build-wasm/checklist.wasm \
   build-wasm/qtloader.js    \
   wasm/
git add wasm/
git commit -m "update WASM build"
git push
```

> **Why singlethread?** GitHub Pages cannot set the `COOP`/`COEP` headers
> required by multithreaded WASM. Single-thread works in all browsers.
